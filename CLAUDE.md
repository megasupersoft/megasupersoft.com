# megasupersoft.com

Corporate website for MegaSuperSoft. Built with VitePress.

## Quick start

```bash
npm install
npm run dev          # local dev server at localhost:5173
npm run build        # production build to .vitepress/dist
```

## Secrets management

Credentials are stored in `.env` (gitignored) and backed up to Cloudflare KV.

```bash
npx wrangler login   # one-time auth
npm run env:pull     # download .env from Cloudflare KV
npm run env:push     # upload .env to Cloudflare KV
```

## Updating social media graphics

```bash
npm run update-socials                  # all platforms
npm run update-socials -- --x           # just X
npm run update-socials -- --bluesky     # just Bluesky
npm run update-socials -- --youtube     # just YouTube (banner only, avatar is Google Account)
npm run update-socials -- --theme light # use light variant assets
```

Requires `.env` with API credentials. See `.env.example` for setup instructions.

## Brand assets

Generated by `screenshot.mjs` using Playwright against the running dev server.

```bash
npm run dev                    # start dev server first
node screenshot.mjs            # regenerate all brand assets
```

Assets live in `public/brand/` with dark/light variants:
- `logo-square-{dark,light}.png` (1024x1024) — profile avatars
- `banner-x-{dark,light}.png` (1500x500) — X/Bluesky banners
- `banner-youtube-{dark,light}.png` (2560x1440) — YouTube banner
- `hero-screenshot-{dark,light}.png` (1920x1080) — promo screenshots

SVG variants are also generated. OG images are in `public/og-image*.png`.

## Project structure

```
index.md, about.md, products.md, ...   # VitePress content pages
.vitepress/config.ts                    # site config, nav, social links, meta
.vitepress/theme/Layout.vue             # custom layout wrapper
.vitepress/theme/index.ts               # theme entry
.vitepress/theme/components/            # Vue components (SiteFooter, etc.)
.vitepress/theme/styles/                # CSS
public/brand/                           # brand asset images (PNG + SVG)
scripts/update-socials.mjs              # push brand assets to social platforms
screenshot.mjs                          # generate brand assets via Playwright
```

## Conventions

- ESM throughout (`"type": "module"` in package.json)
- Zero external dependencies for scripts (use Node built-ins: fetch, crypto, fs)
- Brand assets use dark variant by default
- Playwright is a devDependency (used for screenshot generation)
